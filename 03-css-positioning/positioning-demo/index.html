Understanding different position values and their effects

  • Static
  • Relative
  • Absolute
  • Fixed
  • Sticky

Static Positioning (Default)

Static Box
Static Box
Static Box

Static positioning is the default. Elements follow the normal document flow.

  • Cannot be moved with top, right, bottom, or left properties
  • Elements stack in the order they appear in the HTML
  • Most common positioning type

Relative Positioning

Relative Box
Offset Box
Relative Box

Relative positioning moves an element relative to its normal position.

  • Can be moved using top, right, bottom, or left properties
  • Original space in document flow is preserved
  • Often used as a container for absolutely positioned elements

Absolute Positioning

Absolute Box
Positioned Box
Absolute Box

Absolute positioning removes an element from the normal document flow.

  • Positioned relative to nearest positioned ancestor
  • If no positioned ancestor, uses viewport as reference
  • Commonly used for overlays, tooltips, and modals

Fixed Positioning

Fixed Box

Scroll to see fixed positioning in action!

Fixed Positioned Box

Fixed positioning removes an element from the normal document flow and positions it relative to the viewport.

  • Stays in the same position even when scrolling
  • Commonly used for navigation bars and modals
  • Always positioned relative to viewport
